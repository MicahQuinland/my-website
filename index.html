AWS Cost Report Genereator

Cloud based tool that distrubutes dynamically generated templates

This was a project I worked on to dynamically generate AWS billing report PDFs. All the code was run internally in AWS through Lambda. I wrote several python files that parsed an Excel billing spreadsheet located in S3. This billing file was a cost spreadsheet that was updated daily by AWS. Every month a cloud watch rule would run reports for every AWS account.

For each account, I created a JSON template file which was read into the python script and used to determine what data required for the report. In order to generate the PDF report, I added ReportLab libraries to the Lambda function. ReportLab is an open source reporting tool. The report would then be saved to the local Lambda file system and emailed out to a distribution list using Simple Email Service.

Emergency Evac Tool

Cloud based emergency notification service

This project was a service to alert a list of contacts of an emergency evacuation. This tool allowed a master user to activate an Emergency evacuation through AWS Pinpoint two-way SMS messaging. The account has a designated long-code (a phone number) that once messaged, would be forwarded to Lambda for processing. Lambda then created an AWS Pinpoint campaign from the list of contacts and sent out the alert to their email and phone number.

The user could then reply Safe or Help and their message, phone number, and name would be sent to Lambda and added to a DynamoDB tracking table for those who replied Safe, Help, or have yet to respond. Every two minutes the master user would receive an email from SES with the status of everyone on the contact list. Once the Master user sent the kill text the tool shuts off and resets everything for the next evacuation.

RFID Hex Parser

Improvements to open source RFID API/SDK

Starting from the ThingMagic Mercury API I made modifications to the Java code to meet new requirements of the new RFID implementations. These additions include adding Hex to ASCII conversion, automatic USB port detection, and an alert of the proper data the end user was looking to read from an RFID inlay. Next, I will add range reduction and continuous read to reduce the necessary steps for use and the likelihood of accidental inlay reads.

Autonomous Mini Robot

Robots that battle eachother

While a part of a small team of developers I helped build an autonomous Sumobot from scratch. After first putting together the Sumobot assembly I wrote the C code that orchestrates the robot's decisions. The objective of the Sumobot is to push an opposing bot out of the ring while yourself staying within the boundaries. The Sumobot does not have any controller and make decisions on its own based on sensor data.

I wrote the controller which took input values from two floor facing sensors (Left, Right) and three forward facing sensors (Left, Right, Middle) and turned them into output movements of the (Left, Right) motors either forward or backwards.
